# ProjectX - Server

NestJS 11 backend running on Fastify.

## Running

From the repo root: `pnpm dev` (starts both server and client).

Server only: `pnpm start:dev` (from this directory).

The server loads environment variables from `.env` in this directory. See `.env.example` for all available options.

## Database

Schema files live in `src/db/schema/`, one per domain. Migrations are generated by Drizzle Kit.

```bash
pnpm db:generate add_some_column   # generate migration from schema diff
pnpm db:migrate                     # apply pending migrations
pnpm db:studio                      # open Drizzle Studio (DB browser)
```

## Testing

```bash
pnpm test           # unit tests
pnpm test:watch     # unit tests in watch mode
pnpm test:e2e       # end-to-end tests
pnpm test:cov       # unit tests with coverage
```

## Key conventions

- Constructor injection with `@Inject(DB)` for the Drizzle instance.
- Config via `@nestjs/config` named configs (`appConfig`, `dbConfig`, `authConfig`, `storageConfig`, `mailerConfig`). Never read `process.env` directly in services.
- DTOs use `class-validator` decorators. Global `ValidationPipe` rejects unknown fields.
- Throw NestJS `HttpException` subclasses (`NotFoundException`, `ForbiddenException`, etc.) - never raw `Error`.
- All user-owned data is scoped by `userId`. Service methods check ownership.
